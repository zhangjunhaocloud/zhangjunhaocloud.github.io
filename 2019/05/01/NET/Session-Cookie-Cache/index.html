Session-Cookie-Cache

2019-05-01
  • 原创

Session

用来保存每一个用户的专有信息，生存期是用户持续请求时间加上一段时间(一般是２０分钟)，信息保存在Web服务器内存中，保存数据量可大可小，由于用户停止使用应用程序后它仍在内存中存留一段时间，因此这种方法效率较低
为当前用户会话提供信息。还提供对可用于存储信息的会话范围的缓存的访问，以及控制如何管理会话的方法。它存储在服务器的内存中，因此与在数据库中存储和检索信息相比，它的执行速度更快。与不特定于单个用户会话的应用程序状态不同，会话状态应用于单个的用户和会话。因此，应用程序状态非常适合存储那些数量少、随用户的变化而变化的常用数据。而由于其不发生服务器-客户端数据传输，Session还适合存储关于用户的安全数据，如购物车信息。
Session的关键特性：存储于服务器内存中，与会话有关，在会话的整个生存期中存在即不会被主动丢弃，不被序列化，不发生服务器-客户端数据传输

Cookie

用来保存客户浏览器请求服务器页面的请求信息，我们可以存放非敏感的用户信息，保存时间可以根据需要设置，如果没有设置失效日期，它的生命周期保存到关闭浏览器为止，Cookie对象的Expires属性设置为MinValue表示永不过期.Cookie存储的数据量受限制，大多数浏览器为4k．由于并非所有的浏览器都支持cookie，数据将以明文的形式保存在客户端
Cookie提供了一种在Web应用程序中存储用户特定信息的方法。例如，当用户访问您的站点时，您可以使用Cookie存储用户首选项或其他信息。当该用户再次访问您的网站时，应用程序便可以检索以前存储的信息。在开发人员以编程方式设置Cookie时，需要将自己希望保存的数据序列化为字符串(并且要注意，很多浏览器对Cookie有4096字节的限制)然后进行设置
Cookie的关键特性：存储于客户端硬盘上，与用户相关，在一定时间内持久化存储，可以跨浏览器共享数据，需要被序列化，发生服务器-客户端数据传输

Cache

用于在Http请求期间保存页面或数据，大大提高整个应用程序的效率，Cache实例是每一个应用程序专有的，其生命周期==该应用程序周期
它存储于服务器的内存中，允许您自定义如何缓存项以及将它们缓存多长时间。例如，当缺乏系统内存时，缓存会自动移除很少使用的或优先级较低的项以释放内存。该技术也称为清理，这是缓存确保过期数据不使用宝贵的服务器资源的方式之一。它不与会话相关，所以它是多会话共享的，因此使用它可以提高网站性能，但是可能泄漏用户的安全信息，还由于在服务器缺乏内存时可能被自动移除Cache，因此需要在每次获取数据时检测该Cache项是否存在
Cache的关键特性：存储于服务器内存中，与会话无关，根据服务器内存资源的状况随时可能被丢弃，不被序列化，不发生服务器-客户端数据传输

Session和Cache的区别

  1. 最大的区别是Cache提供缓存依赖来更新数据，而Session只能依靠定义的缓存时间来判断缓存数据是否有效
    即使应用程序终止，只要Cache.Add方法中定义的缓存时间未过期，下次开启应用程序时，缓存的数据依然存在．
  2. 而Session缓存只是存在于一次会话中，会话结束后，数据也就失效了
  3. Session容易丢失，导致数据的不确定性，而Cache不会出现这种情况
  4. 由于Session是每次会话就被加载，所以不适宜存放大量信息，否则会导致服务器的性能降低．而Cache则主要用来存放大容易信息
  5. Session目前只能保存在内存中，对其性能有影响
