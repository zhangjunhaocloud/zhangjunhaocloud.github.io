NET各版本功能

2020-05-21
  • NET

C# 8.0

C# 8.0于2019年9月随着.NET Framework 4.8与VS2019一同发布

  • readonly成员
  • 默认接口成员(重要)
  • 在更多位置中使用更多模式
  • using声明
  • 静态本地函数
  • 可处置的ref结构
  • 可为空引用类型
  • 异步流(重要)
  • 索引和范围

C# 7.3

c# 7.3随着VS2017 v15.7发布于2018年5月

  • 索引fixed字段不需要进行固定
  • 可能会重新分配ref局部变量
  • stackalloc数组支持初始值设定项
  • 更多类型支持fixed语句
  • 增强的泛型约束
  • 元组支持==和!=
  • 将特性添加到自动实现的属性和支持字段
  • in方法重载解析决胜属性
  • 扩展初始值设定项中的表达式变量
  • 改进了重载候选项
  • 公共或开放源代码签名
  • pathmap

C# 7.2

C# 7.2随着VS2017 v15.5发布于2017年11月

  • 安全高效的代码的增强功能
  • 非尾随命名参数
  • 数值文字中的前导下划线

    int binaryValue = 0b_0101_0101;

  • private protected 访问修饰符

    指示可通过包含同一程序集中声明的类或派生类来访问成员。虽然protected internal允许通过同一程序集中的派生或派生类进行访问，但private protected限制对同一程序集中声明的派生类的访问

  • 条件 ref 表达式

C# 7.1

C# 7.1随着VS2017 v15.3发布于2017年8月

  • 异步main方法
  • 默认文本表达式(default)
  • 推断元组元素名称
  • 泛型类型参数的模式匹配
  • 引用程序集生成(-refout、-refonly)

C# 7.0

C# 7.0于2017年3月随着.NET 4.7和VS2017发布

  • out变量(可直接在方法调用的参数列表中声明out变量,而不是编写单独的声明语句)
  • 元组
  • 弃元
  • 模式匹配
  • ref局部变量和返回结果
  • 本地函数
  • 更多的expression - bodied 成员
  • 引发表达式(throw高级应用)

    string arg = args.Length >= 1 ? args[0] : throw new ArgumentException(“You must supply an argument”);

  • 通用的异步返回类型
  • 数字文本语法改进

    public const int Sixteen = 0b0001_0000;

C# 6.0

C# 6.0在2015年7月随着.NET Framework 4.6一同发布，后期发布了.NET Framework 4.6.1、4.6.2
C# 在3.0版和5.0版对面向对象的语言添加了主要的新功能，在6.0版中，它不再推出主导性的杀手锏，而是发布了很多使得C#编程更有效率的小功能。从整体来看，在此版本中，C#消除语言样本，让代码更简洁且更具有可读性

  • 只读自动属性
  • 自动属性初始化表达式
  • Expression-bodied(正文表达式) 函数成员

    public string FullName => $”{FirstName} {LastName}”;

  • 静态导入 using static

    using static System.Console;
    WriteLine(“Hello world”);

  • Null条件运算符(?.)

    一个挺有意思的题目：?、?.、??、?:、?[]分别应用在什么场景下

  • 字符串内插($符号的应用)
  • 异常筛选器(异常捕捉catch中用when)
  • nameof表达式
  • catch和finally块中的await
  • 使用索引器初始化关联集合

    private Dictionary<int, string> webErrors = new Dictionary<int, string>
    {
     [404] = “Page not Found”,
     [302] = “Page moved, but left a forwarding address.”,
     [500] = “The web server can’t come out to play today.”
    };

  • 集合初始值设定项中的扩展Add方法
  • 改进重载解析

C# 5.0

C# 5.0在2012年2月随着.NET Framework 4.5一同发布，后期发布了.NET Framework 4.5.1、4.5.2。5.0版是C#语言有针对性的一个版本。对此版本中所做的几乎所有工作都归于另一个突破性语言概念：适用于异步编程的async和await模型

  • 异步编程(async、await)
  • 调用方信息

C# 4.0

C# 4.0版和Visual Studio 2010一起发布于2010年4月，对应.NET Framework4.0。C# 4.0版很难达到3.0版的创新水平

  • 动态绑定(dynamic)
  • 命名实参和可选实参
  • 泛型中的协变和逆变
  • 嵌入的互操作类型

C# 3.0

C# 3.0版和Visual Studio 2008一起发布于2007年下半年,但完整的语言功能是在.NET Framework 3.5版中发布的。此版本标示着C#发展过程中的重大更改。C#成为真正强大的编程语言

  • 自动实现的属性
  • 匿名类型
  • 隐式类型本地变量
  • 扩展方法
  • 分部方法
  • 对象和集合初始值设定项
  • 查询表达式
  • Lambda表达式
  • LINQ
  • 表达式树(System.Linq.Expression)

C# 2.0

微软于2005年11月发布了.NET Framework 2.0,同时发布了Visual Studio 2005

  • 分部类
  • 泛型
  • 匿名方法
  • 可以为null的类型
  • 迭代器
  • 协变和逆变(协变和逆变能够实现数组类型、委托类型和泛型类型参数的隐式引用转换。协变保留分配兼容性，逆变与之相反)
  • getter/setter单独可访问性

C# 1.2

随着Visual Studio 2003一起提供。它对语法做了一些小改进。最值得注意的是，从此版本开始，当IEnumerator实现IDisposable时,foreach循环中生成的代码会在IEnumerator上调用Dispose

C# 1.0

  • 类
  • 结构
  • 接口
  • 事件
  • 属性
  • 委托
  • 表达式
  • 语句
  • 特性
