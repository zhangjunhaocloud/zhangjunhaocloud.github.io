ASP.NET Web API 2框架揭秘笔记

2020-07-28
  • Web API

  • 七层网络协议(应用层(HTTP、FTP)、表示层、会话层、传输层(TCP、UDP)、网络层(IP、ICMP、IGMP)、链路层、物理层)

  • 位于网络层的IP协议提供的传输方式是不可靠的，它只承诺尽可能地将数据报发送出去，但不能保证发送的数据报能够成功抵达目的地。IP协议的不可靠性还体现在它不能检测数据在传输过程中是否发生了改变，也就是说数据的完整性得不到保证。IP协议是一个无连接的网络协议，每次数据报的处理对它来说均是独立的

  • TCP是一个基于连接的传输协议，数据交换双方在进行报文传输之前需要建立连接，报文传输结束之后需要关闭连接。

  • 常用媒体类型 text/html(HTML格式的文本)、text/xml(application/xml)(XML格式的文本)、text/json(application/json)(JSON格式的文本)、image/gif(image/jpeg、image/png)(GIF、JPEG、PNG格式的图片)、audio/mp4(audio/mpeg、audio/vnd/wave)(MP4、MPEG、WAVE格式的音频文件)、video/mp4(video/mpeg、video/quicktime)(MP4、MPEG、QUICKTIME格式的视频文件)

  • URI、URL和URN

  • 状态码

  • 客户端与WEB服务器在一次HTTP事务中交换的消息被称为HTTP报文，客户端发送给服务器的请求消息被称为请求报文，服务器返回给客户端的响应消息被称为响应报文。完整的HTTP报文由三部分构成：起始行(表示采用的HTTP方法、请求URI和采用的HTTP版本)、报头集合(以空行表示报头集合的结束)、主体内容

  • REST(REpresentational State Transfer) 表现状态转换

  • URL与物理文件紧密绑定在一起的方式的局限性
